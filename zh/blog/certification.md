# 证书备考清单

这是一份未来两年的备考清单。列出来的项目很多，我大概不会全部考完。它更像一张学习地图：当我需要补云计算、Linux、网络或数据分析时，可以找到一条相对完整的课程路线。

证书的价值取决于目标岗位和实际练习。只刷题通过考试，能留下的知识有限；如果把考试大纲当作课程目录，再配合实验和项目，备考过程会更有用。

## 1. 云计算与基础架构

- `AZ-900`：Azure 基础认证，适合了解云服务模型、治理、定价和合规概念。
- `AI-900`：介绍 Azure 上的机器学习与 AI 服务，偏入门。
- `AWS SAA-C03`：覆盖高可用、扩展性、存储、网络和成本设计，适合系统学习 AWS 架构。

如果时间有限，我会先选一个云平台深入，避免同时记忆多套名称相近的产品。

## 2. Linux 与自动化

- `RHCSA`：重点是 Linux 系统管理，包括用户、存储、服务、网络和基础安全。
- `RHCE`：在系统管理基础上继续学习 Ansible 自动化。

这类认证含有较多实操，比纯选择题更能暴露环境配置和命令使用上的薄弱环节。

## 3. 云原生与容器

- `CKA`：Kubernetes 管理员认证，涉及集群、Pod 调度、网络、存储和排障。

在准备 CKA 之前，我需要先熟悉 Linux、容器和基础网络。直接背 `kubectl` 命令，很容易通过一段时间后全部忘掉。

## 4. 网络

- `CCNA / CCNP`：建立 TCP/IP、交换、路由和网络排障的知识框架。
- `HCIA / HCIP`：内容与华为设备和国内项目环境联系更紧。

具体选择要看目标岗位。证书数量本身不会替代抓包、配置设备和定位故障的能力。

## 5. 人工智能与数据分析

- `NVIDIA DLI`：以短课和实验为主，可用于学习 GPU 加速、计算机视觉或自然语言处理。
- `Google Data Analytics Professional Certificate`：覆盖表格、SQL、R 和数据可视化，适合作为数据分析入门。

这些课程更适合补基础。想证明 AI 工程能力，还需要能运行、评估和部署的项目。

## 6. 项目管理

- `PSM I` 或 `CSM`：用于了解 Scrum 的角色、事件和协作方式。

我会把优先级放在与近期目标最接近的认证上，每完成一项都配一个可以展示的实验或项目。这样即使最后没有参加考试，投入的时间也能留下具体成果。
