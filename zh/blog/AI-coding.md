# AI 编程的碎碎念

AI 编程工具更新得很快，产品名称、模型排名和收费方式随时会变。下面写的是我目前的使用感受，重点放在怎么选工具。

## 先分清 IDE 和模型

IDE 是写代码、浏览文件和调试项目的工作环境，模型负责理解要求并生成内容。两者经常被捆在一个产品里出售，也可以分开选择。

这个区别很实用。同一个模型放进不同工具，效果可能差很多，因为工具能看到的文件、可以执行的命令、提示词设计和上下文管理方式都不同。反过来，同一个 IDE 接入不同模型，编程能力也会明显变化。

## 两类常见工具

### 图形界面类

VS Code、Cursor 和 Trae 都属于这一类。它们保留了传统编辑器界面，再加入 AI 对话、代码补全和修改确认。

这类工具容易上手。你可以看到每次改动，在应用前检查差异，适合刚开始接触 AI 编程，或希望对修改保持细致控制的人。

### 终端智能体类

Claude Code、OpenCode 和 Kimi Code 更接近在终端里工作的智能体。它们可以读取项目、执行命令、修改多个文件，并连续完成一组相关任务。

优势来自权限和上下文。智能体能够观察构建结果，再根据报错继续修改，因此更适合排查系统问题或完成跨文件任务。代价也很直接：权限越大，误改范围可能越大。使用者需要会看 diff、读日志，并在关键操作前限制范围。

我以前以为这类工具的门槛在于写出很长的提示词。实际使用后，我更看重任务边界和验收标准。告诉模型哪些文件不能动、结果必须通过什么测试，通常比堆很多形容词有效。

## 模型从哪里来

最常见的接入方式有两种。

一种是直接购买 IDE 自带的订阅，由产品负责模型接入和额度管理。另一种是从模型服务商取得 API Key，再填入支持自定义模型的工具。

API Key 应当放在环境变量或工具的安全配置中，不能提交到公开仓库。接入中转服务时，还要考虑日志留存、密钥管理和代码是否会经过第三方服务器。

## 我对几家模型的使用感受

### Google Gemini

Gemini 适合搜索、长上下文阅读和资料整理。我经常用它先了解一个陌生主题，再核对具体来源。免费额度和可用模型会调整，实际情况以 Google AI Studio 当时的页面为准。

### OpenAI

ChatGPT 是很多人接触生成式 AI 的入口。对编程工作，我更关注 Codex 这类能直接进入项目环境的产品。OpenAI 的优势是产品完整，模型、工具调用和开发平台结合得比较紧。

它的回复风格未必总合我的口味，额度也会影响连续工作。系统提示和项目说明写得越具体，输出通常越稳定。

### Anthropic Claude

Claude 是我处理复杂代码任务时经常选择的模型，配合 Claude Code 尤其顺手。它在阅读大型项目、连续修改和解释代码方面表现稳定。

使用障碍主要来自地区限制、风控和支付。国内用户往往需要额外解决账号与网络问题，这会增加长期使用成本。
