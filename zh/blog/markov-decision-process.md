# 当下一步并不确定：理解马尔可夫决策过程

很多算法都在回答同一个问题：**下一步该怎么走？**

如果地图完全已知，每次行动的结果也完全确定，那么问题通常可以写成一次搜索：从起点出发，找到一条代价最小的路径。但现实很少如此配合。路上可能堵车，设备可能故障，用户可能改变选择；同一个动作在相同条件下，也不一定得到相同结果。

马尔可夫决策过程（Markov Decision Process，MDP）研究的正是这类问题：

> 当行动结果带有随机性，而且今天的选择会影响明天还能做什么时，怎样制定一套长期表现最好的行动规则？

它要在不确定环境中寻找一套**策略**，为每种状态规定下一步行动。

## 一、MDP 到底由什么组成

一个有限 MDP 通常由五部分描述：

- **状态 S**：系统此刻所处的情形。
- **动作 A**：在某个状态下可以做出的选择。
- **转移概率 P(s' | s, a)**：在状态 s 执行动作 a 后，进入状态 s' 的概率。
- **奖励 R(s, a, s')**：这次转移带来的即时收益或代价。
- **折扣因子 γ**：未来奖励相对当前奖励的重要程度，通常满足 0 ≤ γ < 1。

例如，考虑每天从宿舍去教学楼。状态可以包含当前位置、时间、天气和是否临近上课；动作可以是步行、骑车或坐校车。骑车通常最快，但下雨时可能变慢；校车更稳定，却可能需要等待。这里不存在一条永远最优的固定路线，真正需要的是：

- 晴天且时间充足时怎么走；
- 下雨且快迟到时怎么走；
- 错过校车后怎么调整。

这种“看到当前状态，再决定动作”的规则，就是策略 π(a | s)。

“马尔可夫”的含义是：**如果当前状态已经包含决策所需的全部信息，预测下一步时就不必额外回看完整历史。**

如果“当前状态”只写当前位置，却没有包含天气和时间，那么它可能根本不满足马尔可夫性。状态定义不完整，是很多 MDP 模型失效的原因。

## 二、搜索问题与随机环境：路线和策略的区别

搜索算法和 MDP 都可以画成图，但它们解决的问题并不相同。

| 对比 | 搜索问题 | 马尔可夫决策过程 |
| --- | --- | --- |
| 动作结果 | 通常是确定的 | 由概率分布决定 |
| 解的形式 | 一条路径或动作序列 | 每个状态下的行动策略 |
| 评价目标 | 到达目标的路径代价 | 长期累计奖励的期望 |
| 遇到意外 | 往往需要重新规划 | 策略本身就规定了如何应对 |
| 核心运算 | 在候选路径中扩展与比较 | 对未来结果求期望并递推 |

在确定性搜索中，动作 a 往往把状态 s 唯一地带到状态 s'。于是我们可以问：“从起点到终点，哪条路径最短？”

在 MDP 中，同一个动作可能通向多个后继状态。于是问题变成：“考虑每种结果的概率和长期影响，现在选哪个动作的期望最好？”

所以，搜索的答案更像一张预先打印好的路线图；MDP 的答案则更像一份条件化的驾驶规则。路线图告诉你一路向前，策略则告诉你遇到堵车、下雨和封路时分别怎么办。

## 三、策略评估：一条策略究竟有多好

假设策略 π 已经给定。我们首先不讨论怎样改进它，只问一个更基础的问题：

> 从状态 s 出发，以后始终按照 π 行动，平均能得到多少累计奖励？

把从时刻 t 开始的折扣累计奖励记为 Gₜ：

<pre><code>Gₜ = rₜ + γrₜ₊₁ + γ²rₜ₊₂ + γ³rₜ₊₃ + …</code></pre>

状态价值 V^π(s) 就是 Gₜ 在状态 s 下的期望。它满足 Bellman 期望方程：

<pre><code>V^π(s)
= Σₐ π(a | s) Σₛ' P(s' | s, a)
  [R(s, a, s') + γV^π(s')]</code></pre>

这条式子看起来复杂，实际只表达了三层意思：

1. 策略可能以不同概率选择不同动作，因此先对动作求平均；
2. 动作可能产生不同的后继状态，因此再对环境结果求平均；
3. 一次选择的价值，等于即时奖励加上折扣后的未来价值。

这就是 Bellman 递推最重要的思想：**长期价值可以拆成“眼前一步”和“从下一状态继续”的价值。**

### 从 V 值到 Q 值

V^π(s) 评价“处在某个状态有多好”，Q^π(s, a) 则评价“在某个状态先做某个动作，之后再遵循策略有多好”。

它的递推式为：

<pre><code>Q^π(s, a)
= Σₛ' P(s' | s, a)
  [R(s, a, s') + γ Σₐ' π(a' | s')Q^π(s', a')]</code></pre>

二者的关系很直接：

<pre><code>V^π(s) = Σₐ π(a | s)Q^π(s, a)</code></pre>

V 值把策略已经做出的选择平均在一起；Q 值保留了动作维度，因此可以直接比较同一状态下哪个动作更好。强化学习中经常学习 Q 值，原因就在这里：知道了所有 Q(s, a)，就可以选择其中最大的动作。

策略评估的本质，是求解一组相互依赖的方程。在小型问题中可以直接做线性代数求解；状态较多时，通常从任意初值开始反复更新，直到数值稳定。

## 四、价值迭代：一边评估，一边寻找更好的策略

如果策略尚未给定，我们需要在每个状态中选择价值最大的动作。Bellman 期望方程中的“按照策略求平均”，于是变成“选择最大值”：

<pre><code>Vₖ₊₁(s)
= maxₐ Σₛ' P(s' | s, a)
  [R(s, a, s') + γVₖ(s')]</code></pre>

不断应用这次更新，Vₖ 会逐渐逼近最优价值 V*。收敛后，再从价值函数中提取策略：

<pre><code>π*(s)
= arg maxₐ Σₛ' P(s' | s, a)
  [R(s, a, s') + γV*(s')]</code></pre>

一个清晰的同步价值迭代循环可以写成：

<pre><code>初始化所有 V(s)

重复：
    nextV = V 的副本
    delta = 0

    对每个非终止状态 s：
        nextV(s) = 所有动作期望回报的最大值
        delta = max(delta, |nextV(s) - V(s)|)

    V = nextV

直到 delta 足够小</code></pre>

公式只有一行，真正实现时却有几个容易被忽略的工程问题。

### 1. 同步更新还是原地更新

同步更新使用旧的 Vₖ 计算完整的新一轮 Vₖ₊₁，逻辑和理论公式完全一致，也便于复现。

原地更新会立即使用本轮刚算出的新值，类似 Gauss-Seidel 迭代，很多时候更快，但结果会受到状态遍历顺序影响。两者都可以使用，但测试、日志和收敛判断必须与选择的更新方式一致。

### 2. 不要把“变化很小”误当成“误差同样小”

循环中常见的 delta 是相邻两轮的最大变化，而不是当前值与 V* 的真实距离。γ 越接近 1，这两者之间的差距可能越大。

若用 Bellman 残差

<pre><code>residual = ||T(V) - V||∞</code></pre>

作为停止标准，那么可以得到：

<pre><code>||V - V*||∞ ≤ residual / (1 - γ)</code></pre>

因此，希望价值误差不超过 ε 时，一个有保证的充分条件是：

<pre><code>residual ≤ (1 - γ)ε</code></pre>

当 γ = 0.99 时，1 - γ 只有 0.01。看似很小的更新变化，放到长期累计价值上仍可能对应明显误差。

### 3. 终止状态的语义必须统一

如果到达终止状态后不再产生奖励，通常令其价值为 0，并把最后一次奖励放在“进入终止状态”的转移上。若同时给终止状态保留价值，又在转移时发放终止奖励，就可能重复计算。

### 4. 只遍历真实存在的转移

实际 MDP 的转移通常是稀疏的。没有必要对每个 s、a 都扫描全部状态 s'；只遍历概率非零的后继状态，能显著降低时间和内存开销。

## 五、带环图为什么还能收敛

普通图搜索遇到环时，首先想到的是避免重复访问，否则算法可能一直绕圈。MDP 允许状态反复出现：设备可能在“正常、故障、恢复”之间循环，智能体也可能多次回到同一位置。

价值迭代没有因为环而失效，是因为在 γ < 1 时，Bellman 最优算子 T 具有压缩性：

<pre><code>||T(V) - T(W)||∞ ≤ γ||V - W||∞</code></pre>

也就是说，任意两组价值估计经过一次 Bellman 更新后，它们的最大差距至多变成原来的 γ 倍。只要 γ < 1，反复更新就会不断压缩差异，并收敛到唯一的不动点 V*。

环并没有消失，而是被折扣控制了影响。绕得越久，未来奖励乘上的 γ 次方越高，对当前价值的贡献越小。

### 一个明确的迭代次数上界

若每一步奖励的绝对值不超过 Rmax，并从 V₀ = 0 开始，则有：

<pre><code>||Vₖ - V*||∞ ≤ γᵏ Rmax / (1 - γ)</code></pre>

要让误差不超过 ε，只需令：

<pre><code>k ≥ ln[ε(1 - γ) / Rmax] / ln(γ)</code></pre>

最后向上取整即可。

例如 Rmax = 1、ε = 0.01：

- γ = 0.9 时，最坏情况下约需 66 轮；
- γ = 0.99 时，最坏情况下约需 917 轮。

γ 同时影响未来奖励的权重和算法的收敛速度。常说的有效规划视野大约是 1 / (1 - γ)：γ = 0.9 时约为 10 步，γ = 0.99 时则约为 100 步。

这个上界通常比较保守，但它给出了可验证的最坏情况保证。

### γ = 1 时要格外小心

如果 γ = 1，压缩性不再自动成立。在含有正奖励的循环中，累计价值甚至可能无限大；即使奖励有界，价值迭代也未必按上述方式收敛。

γ = 1 并非永远不可用。有限时域任务，或能够保证最终终止的随机最短路问题，可以在额外条件下处理。但此时不能继续套用 γ < 1 的收敛结论，必须单独证明终止性和价值的有界性。

## 六、真正需要记住的四件事

马尔可夫决策过程并不神秘，它只是把“做选择”这件事拆成了四个更精确的问题：

1. **搜索与决策不同**：确定性搜索寻找路线，随机环境需要策略。
2. **价值来自递推**：当前价值等于即时奖励与未来价值的组合。
3. **Q 值保留动作信息**：它让我们能在同一状态下直接比较不同选择。
4. **收敛依赖压缩性**：带环并不可怕，真正关键的是折扣、有界奖励和正确的停止标准。

从最短路径到机器人控制，从库存管理到推荐系统，表面上它们相距很远，底层却都可能是同一个问题：我们无法确定下一步会发生什么，但仍然必须现在做出选择。

MDP 为多种可能到来的未来，提前准备了一套可执行的规则。
