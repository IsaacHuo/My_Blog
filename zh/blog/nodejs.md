# 什么是 Node.js？从原理到应用

Node.js 是基于 V8 的 JavaScript 运行时。它让 JavaScript 可以离开浏览器，访问文件、网络、进程等操作系统能力，因此常被用来编写服务器和命令行工具。

理解 Node.js 时，事件循环、异步 I/O 和工作线程比“单线程”这个标签更重要。

## 1. 异步非阻塞 I/O

程序读取文件或等待网络响应时，CPU 通常没有计算工作可做。如果一个线程始终停在原地等待，大量并发连接会带来线程和上下文切换开销。

Node.js 会把许多 I/O 操作交给操作系统或 libuv。主线程可以继续处理其他任务，结果准备好后，再由事件循环安排对应的回调或 Promise 后续逻辑。

“非阻塞”不等于所有代码都会自动并行。长时间运行的 JavaScript 计算仍会占住主线程，其他回调只能等待。

## 2. 事件循环

事件循环不断从不同阶段取出已经就绪的任务，例如计时器回调、I/O 回调和关闭事件。微任务队列中的 Promise 回调也有自己的执行时机。

这个模型适合大量连接和频繁 I/O。它也要求开发者避免在主线程执行耗时计算，并正确处理异步错误、超时和资源释放。

## 3. Node.js 真的是单线程吗

JavaScript 默认在一个主线程上执行，但 Node.js 进程内部并非只有一个线程。libuv 会使用线程池处理部分文件系统、DNS 和加密任务，V8 自身也有后台线程。应用还可以使用 `worker_threads` 或多进程处理 CPU 密集型工作。

所以更准确的说法是：Node.js 默认用单个事件循环线程执行 JavaScript，并通过系统异步能力、线程池和额外工作线程完成其他任务。

## 4. 架构组成

1. V8：解析、编译并执行 JavaScript。
2. libuv：提供跨平台事件循环、异步 I/O 和线程池。
3. 核心模块：`http`、`fs`、`path`、`net` 等模块暴露常用系统能力。

## 5. 常见用途

- REST、GraphQL 和 WebSocket 服务。
- BFF 与 API 聚合层。
- 命令行工具和前端构建工具。
- Electron 桌面应用。
- I/O 密集的任务处理与自动化脚本。

视频编码、大规模科学计算等 CPU 密集任务需要工作线程、独立服务或更合适的语言实现，否则会拖慢事件循环。
