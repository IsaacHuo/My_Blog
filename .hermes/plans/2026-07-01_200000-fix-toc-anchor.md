# 正文-TOC 间距根治方案

> **For Hermes:** 直接执行。

**Goal:** 将 TOC 从"固定距视口右边缘"改为"固定距正文右边缘"，确保任意视口宽度下间距一致

**根因:** 当前 TOC 使用 `position: fixed; right: 2rem`——锚定视口右边缘，不感知正文位置。视口变窄时正文（flex居中）向 TOC 靠拢，间距不稳定。之前两次调整（加 padding、缩正文宽度）都只缓解特定视口，没有解决根本问题。

**Architecture:** 将 `right` 值从固定 `2rem` 改为 `calc(50vw - 320px - 200px - 3rem)`，其中 `320px` 是正文半宽（640/2），`200px` 是 TOC 宽度，`3rem` 是目标间距。这样 TOC 始终跟随正文右边缘外扩 3rem。

**Tech Stack:** CSS calc()

### Task 1: 修改 TOC right 为 calc 表达式

**Objective:** 将 TOC 定位从视口锚定改为正文锚定

**Files:**
- Modify: `.vitepress/theme/custom.css:222`

**Step 1: 替换 right 值**

```css
  /* Right Sidebar (TOC) positioning */
  .vp-doc aside,
  .vp-doc .aside {
    position: fixed !important;
    right: max(1rem, calc(50vw - 320px - 200px - 3rem)) !important;
    top: 50% !important;
    transform: translateY(-50%) !important;
    width: 200px !important;
    z-index: 10 !important;
    padding-left: 0 !important;
  }
```

说明：
- `50vw - 320px` = 正文右边缘到视口中心的距离（正文半宽）
- `- 200px` = 减去 TOC 自身宽度
- `- 3rem` = 目标间距 48px
- `max(1rem, ...)` = 窄屏下至少保留 16px 边距，TOC 不会被挤出屏幕
- `padding-left: 0` = 之前加的内边距不再需要（间距由 calc 保证）

**Step 2: 同时把之前加的 padding-left 归零**

因为间距现在由 calc 中的 `3rem` 保证，TOC 内部的 `padding-left: 2.5rem` 反而会让文字偏右，归零即可。

### 风险与回退

- `calc()` 依赖 `100vw`，在 iOS Safari 上可能有滚动条宽度偏差（约 15px），但影响很小（间距偏差 < 15px）
- 如果将来正文宽度再次调整，需同步改 `320px` 为新的半宽值
- 回退：`git revert` 即可恢复 `right: 2rem`
